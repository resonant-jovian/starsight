# Rust 2024 / starsight idioms

- **MSRV 1.89, Edition 2024** — workspace-pinned. Don't introduce features that require a newer toolchain or edition. New bits gated on `cfg(feature = "...")` only when the dependency is itself optional.
- **Errors**: `crate::errors::{Result, StarsightError}`. Public APIs return `Result<T>` (the alias). Never `crate::error::*` — that path does not exist.
- **No `unwrap()` / `expect()` in library code** (`src/`). Tests, examples, doctests are fine. Library code propagates with `?` or maps via `.map_err(StarsightError::…)`.
- **Workspace dependencies** live in the root `Cargo.toml` `[workspace.dependencies]` table. Member crates use `dep = { workspace = true }`. Don't pin a version in a member if it already exists at the workspace level.
- **Module path style**: `crate::paths::Path` (the project's `Path` type), not `std::path::Path`. If you need the std one, qualify it: `std::path::Path`.
- **Casts**: prefer `f64::from(x)` (lossless conversions) over `x as f64` per the existing snapshot-test style. Use `as` only for narrowing where wrap-on-overflow is intended.
- **Public items** must have rustdoc — CI runs with `-D missing-docs -D rustdoc::broken-intra-doc-links`. A one-line `///` is enough; doctests are encouraged but not mandatory.
- **No global state.** No `static mut`, no `OnceCell` defaults that bake in user-visible config. Builders, not singletons.
